# USER.md — Supervisor

## Profil Utilisateur Cible

### Audience Primaire

- **Profil** : Grand public curieux, journalistes, étudiants, analystes
- **Niveau d'expertise** : Variable — l'interface doit être accessible à tous niveaux
- **Attente principale** : Comprendre un sujet complexe à travers une expérience immersive et fiable

### Critères de Satisfaction Utilisateur

1. **Compréhension** — L'utilisateur quitte l'expérience en ayant compris la chronologie et les enjeux du sujet.
2. **Confiance** — Les sources sont visibles et vérifiables. L'utilisateur sait que l'information est fiable.
3. **Engagement** — L'utilisateur explore activement (scroll, clicks, comparaisons) plutôt que de consommer passivement.
4. **Fluidité** — Aucune saccade, aucun temps de chargement perceptible, transitions naturelles.
5. **Mémorisation** — Les animations et la structure narrative facilitent la rétention d'information.

### Ton Global de l'Expérience

- **Éducatif** sans être académique
- **Immersif** sans être fatigant
- **Précis** sans être austère
- **Interactif** sans être gadget

### Niveaux de Lecture

| Niveau | Usage | Composant |
|--------|-------|-----------|
| Survol | Parcourir rapidement la chronologie | Timeline + ProgressBar |
| Exploration | Comprendre les événements clés | FocusMode |
| Analyse | Comparer et trouver des corrélations | TimelineComparison |
| Investigation | Vérifier les sources et croiser les données | FocusMode + sources |

### Langues

- **Primaire** : Français
- **Secondaire** : Anglais (prêt pour internationalisation)
